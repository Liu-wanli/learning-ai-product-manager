STYLE 01 / 05

苹果官网极简风

Apple-inspired Minimalist Website
适合企业官网、科技产品、课程介绍、个人品牌和作品集。核心是大面积留白、精准对齐、超大标题和克制动效。

中文完整 Prompt

你是一名资深 UI/UX 设计师和前端开发工程师。 请为我设计并开发一个具有苹果官网视觉气质的极简网页，并输出完整、可直接运行的 HTML、CSS 和 JavaScript 代码。 【整体目标】 网页需要简洁、高级、克制、现代，具有强烈的品牌感和产品感。不要做成普通模板网站，也不要堆砌装饰和颜色。 【视觉风格】 1. 使用白色或非常浅的灰色作为主要背景。 2. 页面保留大面积留白，让每个模块都有充足的呼吸感。 3. 所有内容需要严格居中、对齐，并遵循统一的网格系统。 4. 首页使用超大标题、简短副标题和一个核心视觉区域。 5. 主色控制在黑色、白色、灰色，可以少量使用蓝色作为强调色。 6. 卡片使用大圆角、极轻的边框和柔和阴影。 7. 字体清晰、现代，标题与正文具有明确层级。 8. 避免廉价渐变、复杂纹理、花哨图标和过多颜色。 【页面结构】 请至少包含： - 顶部半透明悬浮导航栏 - 首页 Hero 大标题区域 - 一个轮播图或大型视觉展示区域 - 三张核心特色卡片 - 产品、作品或项目展示模块 - 数据或优势展示模块 - 行动按钮区域 - 简洁页脚 【交互与动画】 1. 页面滚动时，文字、图片和卡片缓慢淡入并向上出现。 2. 鼠标经过卡片时，卡片轻微上浮和放大。 3. 按钮悬停时出现轻微阴影和位移动画。 4. 导航栏滚动时保持固定，并带有毛玻璃效果。 5. 所有动画持续时间控制在 0.4 至 0.8 秒。 6. 动画需要自然、克制，不要出现夸张弹跳。 【技术要求】 1. 使用原生 HTML、CSS 和 JavaScript。 2. 不使用 Bootstrap、Tailwind、Vue 或 React。 3. 所有代码结构清晰，并添加必要注释。 4. 页面必须适配电脑、平板和手机。 5. 保证代码复制后可以直接运行。 6. 如果缺少真实图片，可以使用纯色渐变块或占位图代替。 7. 最终请分别输出完整的 index.html、style.css 和 script.js 代码。 请先思考整体页面结构和视觉层级，再开始输出代码。

English Full Prompt

You are a senior UI/UX designer and front-end engineer. Design and develop a premium Apple-inspired minimalist website, and provide complete, production-ready HTML, CSS, and JavaScript code. [Overall Goal] The website should feel minimal, refined, calm, modern, and highly polished. It must not look like a generic template. Avoid visual clutter, unnecessary decoration, and excessive colors. [Visual Direction] 1. Use a white or very light-gray background. 2. Create generous whitespace between all sections. 3. Keep all content precisely aligned and based on a consistent grid system. 4. Use oversized typography, concise copy, and one dominant visual focus in the hero section. 5. Keep the main palette limited to black, white, and gray, with a small amount of blue as an accent. 6. Use large rounded corners, subtle borders, and very soft shadows. 7. Create a clear hierarchy between headings, subheadings, and body text. 8. Avoid cheap gradients, busy textures, decorative icons, and excessive colors. [Page Structure] Include at least: - A floating translucent navigation bar - A large hero section - A carousel or large visual showcase - Three core feature cards - A product, project, or portfolio showcase - A metrics or advantages section - A call-to-action section - A clean footer [Interactions and Motion] 1. Reveal text, images, and cards slowly as the user scrolls. 2. Add subtle card lift and scale effects on hover. 3. Add gentle shadow and movement feedback to buttons. 4. Keep the navigation fixed and apply a glass-like blur effect. 5. Keep animation durations between 0.4 and 0.8 seconds. 6. Motion should feel natural and restrained, with no exaggerated bouncing. [Technical Requirements] 1. Use vanilla HTML, CSS, and JavaScript. 2. Do not use Bootstrap, Tailwind, Vue, or React. 3. Keep the code clean, well-structured, and commented where needed. 4. Make the website fully responsive for desktop, tablet, and mobile. 5. Ensure the code runs immediately after being copied. 6. Use gradient blocks or placeholders when real images are unavailable. 7. Output complete index.html, style.css, and script.js files separately. Think through the content hierarchy and visual structure before writing the code.
