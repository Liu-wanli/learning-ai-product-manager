Day02｜新建GitHub仓库并发布个人网站完整流程

今天记录从零创建一个 GitHub 网站仓库、上传网站文件、开启 GitHub Pages、补充自动更新流程，并测试新笔记是否能自动出现在网站里的完整过程。

一、目标

我要搭建一个长期使用的个人学习网站，用来沉淀 AI 产品经理学习过程中的内容。

这个网站需要支持：

  • 首页作为学习看板，只展示核心板块入口。
  • 每个板块都有自己的列表页。
  • 上传新 HTML 内容后，对应列表可以自动更新。
  • 网站通过 GitHub Pages 免费发布。

二、整体流程

  1. 新建 GitHub 仓库。
  2. 上传网站项目文件。
  3. 补充 GitHub Actions 自动化文件。
  4. 开启 GitHub Pages。
  5. 等待 Actions 和 Pages 执行完成。
  6. 打开网站链接验证是否发布成功。
  7. 上传一篇新笔记，测试自动更新是否正常。

三、新建 GitHub 仓库

第一步是在 GitHub 上创建一个新的仓库，用来保存网站的所有文件。

  1. 打开 GitHub 首页。
  2. 点击右上角的加号。
  3. 选择 New repository。
  4. 填写仓库名称，例如 learning-ai-product-manager。
  5. 选择 Public。
  6. 不要勾选 Add a README file。
  7. 点击 Create repository。

小白解释

仓库可以理解成一个网上文件夹。这个文件夹专门放网站文件。选择 Public 是为了让 GitHub Pages 可以公开访问网站。

不勾选 README，是为了让仓库保持空白，后面直接上传完整网站包，避免文件重复或冲突。

四、上传网站文件

仓库创建好后，需要把网站文件上传进去。

  1. 点击 uploading an existing file。
  2. 解压网站项目压缩包。
  3. 把解压后的所有文件拖到上传页面。
  4. 确认文件列表里有 index.html、style.css、app.js、section.js 等核心文件。
  5. 点击 Commit changes。

小白解释

GitHub 不能直接运行压缩包，所以不能上传 zip 本身。必须先解压，再上传里面的文件。

index.html 是网站首页，style.css 决定网站好不好看，app.js 和 section.js 负责页面里的自动展示逻辑。

五、检查是否缺少关键文件

上传后，要检查仓库里是否有这些文件和文件夹：

  • index.html：首页
  • style.css：样式
  • app.js：首页逻辑
  • section.js：板块列表页逻辑
  • data/content.json：内容目录数据
  • sections/：五个板块的列表页
  • notes/：每日学习笔记
  • english/：今日英语
  • projects/：工作项目
  • ai-practice/：AI实操
  • learning-camp/：Learning Camp
  • scripts/generate-content-list.js：自动生成内容目录的脚本
  • .github/workflows/generate-content-list.yml：GitHub Actions 自动化流程

小白解释

如果没有 .github/workflows 文件，网站也能打开，但上传新文章后不会自动更新目录。

.github 这个文件夹有时候无法通过网页拖拽上传，需要手动创建。

六、补充 GitHub Actions 文件

如果仓库里没有 .github/workflows，就需要手动创建自动化文件。

  1. 点击 Add file。
  2. 点击 Create new file。
  3. 文件名输入 .github/workflows/generate-content-list.yml。
  4. 粘贴自动化配置内容。
  5. 点击 Commit changes。

小白解释

GitHub Actions 就像一个自动小助手。每次我上传新的学习笔记、英语单词或项目内容，它会自动运行脚本，重新生成内容目录。

这个目录就是 data/content.json。网站列表页会读取它，所以内容才能自动出现。

七、开启 GitHub Pages

文件上传完成后，要开启 GitHub Pages，让仓库变成真正的网站。

  1. 进入仓库页面。
  2. 点击 Settings。
  3. 左侧找到 Pages。
  4. Source 选择 Deploy from a branch。
  5. Branch 选择 main。
  6. Folder 选择 /root。
  7. 点击 Save。

小白解释

GitHub 仓库只是存文件的地方。GitHub Pages 是把这些文件发布成网站的功能。

选择 main 和 /root，意思是：请 GitHub 从 main 分支的根目录读取网站首页。

八、查看网站链接

开启 Pages 后，GitHub 会生成一个网站链接。

如果仓库名是 learning-ai-product-manager，网站地址通常是：

https://liu-wanli.github.io/learning-ai-product-manager/

刚开启时可能会显示 404，通常等待 1 到 3 分钟，再刷新页面即可。

小白解释

404 不一定代表失败，很多时候只是网站还在发布中。等 GitHub 处理完，就能正常打开。

九、测试自动更新

网站能打开后，可以上传一篇新笔记测试自动更新。

  1. 进入仓库里的 notes 文件夹。
  2. 点击 Add file。
  3. 上传新的 HTML 笔记，例如 day02.html。
  4. 点击 Commit changes。
  5. 进入 Actions 页面，等待流程变成绿色对勾。
  6. 刷新网站里的每日学习笔记列表页。
  7. 如果能看到 Day02，说明自动更新成功。

小白解释

上传 day02.html 后，Actions 会扫描 notes 文件夹，发现多了一篇文章，就会更新 data/content.json。

网站的列表页读取新的 data/content.json，就能显示这篇新笔记。

十、常见问题

1. 为什么我看不到 .github 文件夹？

.github 是以点开头的特殊文件夹，网页上传时有时不容易传上去。可以用 Create new file 的方式手动创建完整路径。

2. 为什么网站打开是 404？

可能是 Pages 还没发布完成，也可能是仓库没有 index.html。先等几分钟，再检查 Settings → Pages 是否设置正确。

3. 为什么上传新笔记后列表没有更新？

先看 Actions 是否变成绿色对勾。如果没有 Actions 文件，或者 Actions 失败，目录就不会自动更新。

4. 我以后要改网站，应该改哪里？

  • 改首页：修改 index.html。
  • 改样式：修改 style.css。
  • 改板块列表页：修改 sections 文件夹里的页面。
  • 改自动更新逻辑：修改 scripts/generate-content-list.js。
  • 新增内容：上传 HTML 到 notes、english、projects、ai-practice 或 learning-camp。

十一、今天的总结

今天我完成了从新建仓库到发布网站的完整流程，也理解了 GitHub 仓库、GitHub Pages 和 GitHub Actions 的分工。

  • GitHub 仓库负责保存网站文件。
  • GitHub Pages 负责把网站发布到互联网。
  • GitHub Actions 负责自动更新内容目录。
  • 我以后只需要上传新内容，网站就可以持续成长。
