Day03｜新建GitHub仓库和上传网站文件操作笔记

这篇笔记记录我今天和 Codex 一起完成的新网站搭建流程。重点不是讲复杂原理，而是把每一步点哪里、选什么、注意什么写清楚，方便以后自己重复操作。

一、我要做什么

我要新建一个 GitHub 仓库，把 Codex 生成的网站文件上传进去，然后开启 GitHub Pages，让它变成一个可以访问的网站。

最终目标是：

  • GitHub 仓库负责保存网站文件。
  • GitHub Pages 负责发布网站。
  • GitHub Actions 负责以后上传新内容时自动更新列表。

二、新建仓库的具体操作

  1. 打开 GitHub 网站。
  2. 点击右上角的加号按钮。
  3. 选择 New repository。
  4. 在 Repository name 里填写仓库名字。
  5. 选择 Public。
  6. 不要勾选 Add a README file。
  7. 点击绿色按钮 Create repository。

小白解释

仓库就像一个放在 GitHub 上的文件夹。新建仓库，就是给网站准备一个专门的存放位置。

选择 Public，是因为我要把网站公开发布出来。如果仓库是 Private，有些发布功能可能会受限制。

不勾选 Add a README file，是因为 Codex 给我的网站包里已经有 README.md。如果这里先让 GitHub 创建一个 README，后面上传文件时可能会重复或冲突。

三、新仓库创建完成后看到的页面

仓库新建成功后，GitHub 会显示一个空仓库页面。页面里会有一块蓝色区域，里面有这句话：

Get started by creating a new file or uploading an existing file.

这时候我不用管下面那些命令行代码，因为我现在是用网页上传文件，不需要命令行。

我要点哪里

点击蓝色区域里的：

uploading an existing file

小白解释

这一步的意思是：我要把电脑里的网站文件上传到 GitHub 仓库里。

四、上传网站文件的具体操作

  1. 先找到 Codex 给我的网站压缩包。
  2. 先解压压缩包。
  3. 打开解压后的文件夹。
  4. 把里面的文件和文件夹拖到 GitHub 上传页面。
  5. 确认上传列表里出现了网站文件。
  6. 页面下方点击 Commit changes。

小白解释

不能直接上传 zip 压缩包，因为 GitHub Pages 需要读取真正的网站文件，比如 index.html、style.css，而不是读取压缩包。

Commit changes 可以理解成“确认保存这次上传”。点了以后，这些文件才真正进入仓库。

五、上传后应该有哪些文件

上传完成后，仓库里应该能看到这些核心文件和文件夹：

  • index.html：网站首页。
  • style.css：网站样式。
  • app.js：首页展示逻辑。
  • section.js：板块列表页展示逻辑。
  • data/content.json：网站内容目录。
  • sections/：每日学习笔记、AI实操、Learning Camp、工作项目、今日英语这几个板块的列表页。
  • notes/：每日学习笔记内容。
  • english/：英语词汇内容。
  • projects/：工作项目内容。
  • ai-practice/：AI实操内容。
  • learning-camp/：Learning Camp 内容。
  • scripts/generate-content-list.js：自动生成内容目录的脚本。

六、最容易漏掉的文件：.github

今天操作时发现，网页上传文件时，经常会漏掉 .github 文件夹。

如果仓库里没有这个文件：

.github/workflows/generate-content-list.yml

那么网站虽然可以打开，但以后上传新笔记时，列表不会自动更新。

怎么补 .github 文件

  1. 回到仓库首页。
  2. 点击 Add file。
  3. 点击 Create new file。
  4. 文件名输入：.github/workflows/generate-content-list.yml
  5. 把 Codex 给的 Actions 内容粘进去。
  6. 点击 Commit changes。

小白解释

GitHub 会根据文件名里的斜杠，自动帮我创建文件夹。所以我不用先创建 .github 文件夹，只要直接输入完整路径就可以。

Actions 文件就像一个自动任务说明书。它告诉 GitHub：当我上传新内容时，要自动运行脚本，更新网站目录。

七、开启 GitHub Pages 的具体操作

  1. 进入仓库页面。
  2. 点击顶部菜单里的 Settings。
  3. 在左边找到 Pages。
  4. Source 选择 Deploy from a branch。
  5. Branch 选择 main。
  6. Folder 选择 /root。
  7. 点击 Save。

小白解释

GitHub Pages 是 GitHub 提供的免费网站发布功能。

选择 main 和 /root 的意思是：让 GitHub 从 main 分支的最外层目录里找 index.html，然后把它发布成网站。

八、去哪里看网站链接

开启 GitHub Pages 后，仍然在 Settings → Pages 页面查看网站链接。

如果仓库名字是 learning-ai-product-manager，那么网站地址通常是：

https://liu-wanli.github.io/learning-ai-product-manager/

小白解释

链接格式通常是：

https://用户名.github.io/仓库名/

如果刚打开是 404，不一定是失败，可能是 GitHub 还在发布。等 1 到 3 分钟再刷新。

九、测试自动更新的具体操作

网站能打开后，我可以上传一篇新笔记，测试列表是否会自动更新。

  1. 进入仓库里的 notes 文件夹。
  2. 点击 Add file。
  3. 选择 Upload files。
  4. 上传新的笔记文件，例如 day03.html。
  5. 点击 Commit changes。
  6. 点击仓库顶部的 Actions。
  7. 等待最新流程变成绿色对勾。
  8. 刷新网站里的每日学习笔记列表页。

小白解释

绿色对勾表示 GitHub 自动任务运行成功。

如果列表页出现了新上传的笔记，就说明自动更新成功。

十、今天记住的几个关键词

  • Repository：仓库，保存网站文件的地方。
  • Commit changes：提交修改，把上传或编辑的内容保存到仓库。
  • GitHub Pages：把仓库里的网页文件发布成网站。
  • GitHub Actions：自动执行任务，比如自动更新内容目录。
  • index.html：网站首页。
  • style.css：网站样式。
  • data/content.json：网站各板块的内容目录。

十一、以后我更新网站时怎么做

  • 新增学习笔记：上传到 notes 文件夹。
  • 新增英语单词：上传到 english 文件夹。
  • 新增工作项目：上传到 projects 文件夹。
  • 新增 AI 实操：上传到 ai-practice 文件夹。
  • 新增 Learning Camp 内容：上传到 learning-camp 文件夹。

上传完成后，去 Actions 看绿色对勾，再刷新网站。

十二、今日总结

今天我学会了从零新建 GitHub 仓库、上传网站文件、补充自动化文件、开启 GitHub Pages，并知道了在哪里查看网站链接。

这套流程以后可以重复使用：先建仓库，再上传文件，再开启 Pages，最后用 Actions 测试自动更新。
